**1. Розроблена сутність 3**

Сутність **Stream** призначена для збереження інформації про прослуховування пісень та містить такі поля:

* **user_id** — ідентифікатор користувача, який прослуховував пісню

* **song_id** — ідентифікатор пісні, що була прослухана

* **played_at** — дата та час прослуховування

**2. Інструкція збірки та запуску проекту**

2.1. Встановлення залежностей

Перед першим запуском необхідно встановити всі бібліотеки (включаючи Jest для тестів):

    npm install
    
2.2. Запуск Додатка

Команда запуску додатку та mongoDb:

    docker-compose up --build

Додаток буде доступний на http://localhost:7777.

**3. Інструкція запуску тестів**
Запуск тестів

Команда запуску тестів:

    npm test

4. API Ендпоінти

| Метод | Шлях | Опис |
| :--- | :--- | :--- |
| POST | /api/stream | Створити запис про прослуховування. |
| GET | /api/stream | Отримати історію прослуховувань за ID об'єкта. |
| PUT | /api/stream/_count | Отримати агреговану кількість стримів для списку ID. |
